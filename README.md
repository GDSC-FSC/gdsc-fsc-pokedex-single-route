### Adding New Pages/Routes
With the App Router, you define new routes by creating folders inside the `src/app/` directory. Each folder can contain a `page.tsx` file that exports a React component.

**Example: Adding a Pokedex Detail Page (`/pokemon/1`)**

1.  Create a new folder `src/app/pokemon/[id]`. The `[id]` signifies a dynamic route segment.
2.  Inside `src/app/pokemon/[id]`, create a `page.tsx` file:

    ```typescript jsx
    // src/app/pokemon/[id]/page.tsx
    import { Suspense } from 'react';

    interface PokemonDetailPageProps {
      params: { id: string };
    }

    // This could be a Server Component or Client Component
    async function getPokemonDetails(id: string) {
      // In a real app, you'd fetch data here
      // For now, simulate with a delay
      await new Promise(resolve => setTimeout(resolve, 1000));
      return {
        id,
        name: `Pikachu (ID: ${id})`,
        type: 'Electric',
        description: `This is a placeholder for Pokemon ID ${id}.`,
      };
    }

    export default async function PokemonDetailPage({ params }: PokemonDetailPageProps) {
      const pokemon = await getPokemonDetails(params.id);

      return (
        <div className="min-h-screen flex flex-col items-center justify-center p-8 bg-gray-100 dark:bg-gray-900 text-gray-900 dark:text-white">
          <h1 className="text-4xl font-bold mb-4">Pokemon: {pokemon.name}</h1>
          <p className="text-xl mb-2">Type: {pokemon.type}</p>
          <p className="text-lg text-center max-w-prose">{pokemon.description}</p>
          <a href="/" className="mt-8 text-blue-500 hover:underline">Back to Home</a>
        </div>
      );
    }
    ```
3.  Now, navigate to `http://localhost:3000/pokemon/1` (or any ID) in your browser to see your new dynamic route!

## üöß Limitations, Known Issues & Future Roadmap

This project is intended as a starting point. As such, it has certain limitations and a clear path for future enhancements.

### Current Limitations
*   **Single Route Focus**: Currently, the application is primarily a boilerplate for a single route. It lacks complex navigation or multiple distinct pages.
*   **No Data Fetching**: No API integration or data fetching logic is implemented out-of-the-box. The `getPokemonDetails` function in the example is a mock.

*   **No Authentication/Authorization**: This is a purely public-facing application.

### ‚ùì FAQ

<details>
<summary><b>How do I add a new page (route)?</b></summary>
With the Next.js App Router, you add new pages by creating a new folder inside the `src/app/` directory. Inside that folder, create a `page.tsx` file that exports a React component. For dynamic routes, use square brackets like `[slug]` or `[id]` in the folder name. For example, `src/app/about/page.tsx` creates the `/about` route, and `src/app/pokemon/[id]/page.tsx` creates a dynamic route like `/pokemon/123`.
</details>

<details>
<summary><b>How can I fetch data in Next.js 14?</b></summary>
Next.js 14 with the App Router supports several data fetching methods:
*   **Server Components**: You can use `async/await` directly in your Server Components (e.g., `page.tsx`, `layout.tsx`) to fetch data. Next.js automatically memoizes fetches with the same URL.
*   **Client Components**: Use standard React fetching patterns like `useEffect` with `useState`, or a library like React Query or SWR for client-side data fetching.
*   **Route Handlers**: Create API endpoints in `src/app/api/your-route/route.ts` to handle server-side logic and data fetching, which can then be called by Client Components.

</details>

<details>
<summary><b>How do I customize the Tailwind CSS theme?</b></summary>
You can customize the Tailwind CSS theme by modifying `tailwind.config.ts`. The `theme.extend` object allows you to add or override default colors, fonts, spacing, and more without losing Tailwind's base utilities. For example, to add a new primary color:
```typescript
// tailwind.config.ts
import type { Config } from "tailwindcss";

const config: Config = {
  // ...
  theme: {
    extend: {
      colors: {
        primary: '#FF6347', // Custom primary color
      },
      // ... other extensions
    },
  },
  plugins: [],
};
export default config;
```
</details>

*   **Node.js Version**: Ensure your Node.js version meets the prerequisites (18.x or higher). You can use a tool like `nvm` (Node Version Manager) to switch Node.js versions.
*   **Cache Issues**: Try clearing your npm cache: `npm cache clean --force` and then `rm -rf node_modules package-lock.json` before running `npm install` again.
*   **Permissions**: On some systems, you might encounter permission errors. Try running `npm install` with elevated privileges (e.g., `sudo npm install` on Linux/macOS, or open your terminal as Administrator on Windows).
</details>

<details>
<summary><b>"Port 3000 is already in use" error</b></summary>
This means another process is already using the default port for Next.js.
*   **Identify the process**: On Linux/macOS, use `lsof -i :3000`. On Windows, `netstat -ano | findstr :3000`.
*   **Kill the process**: Terminate the process using the identified PID. On Linux/macOS, `kill -9 <PID>`. On Windows, `taskkill /PID <PID> /F`.
*   **Use a different port**: You can also start Next.js on a different port: `npm run dev -- -p 3001` (or `PORT=3001 npm run dev`).
</details>

<details>
<summary><b>Styling issues with Tailwind CSS</b></summary>
*   **`content` path**: Ensure that the `content` array in `tailwind.config.ts` correctly points to all files where you are using Tailwind classes. If files are not included, Tailwind won't generate the necessary CSS.
*   **`@tailwind` directives**: Verify that `src/app/globals.css` contains the `@tailwind` directives:
    ```css
    @tailwind base;
    @tailwind components;
    @tailwind utilities;
    ```
*   **Restart development server**: Sometimes, especially after modifying `tailwind.config.ts` or `globals.css`, you need to restart the `npm run dev` server for changes to take effect.
</details>
